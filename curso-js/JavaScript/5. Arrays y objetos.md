## Spread operator

Expandimos un  array en base a todos  sus  elementos. Es  muy  c√≥modo para no andar refiri√©ndonos a cada uno de los elementos del array o recorrerlos con un bucle.


***Spread operator***: Saca todos los elementos del array y no los almacena en una variable.
***Destructuring***: Saca determinados elementos del array y los almacena en una variable

## Rest pattern


El elemento rest  siempre es el √∫ltimo y solo puede haber uno por asignaci√≥n, ya que tiene un n√∫mero  variable de elementos que no se  puede definir.

Podr√≠amos haber  definido la funci√≥n add con un  argumento de tipo array, pero observa que as√≠ puede recibir  tanto varios argumentos como un array ya que el spread es sencillo y  queda una funci√≥n m√°s  vers√°til. Es frecuente ver este tipo de l√≥gica en las funciones.
