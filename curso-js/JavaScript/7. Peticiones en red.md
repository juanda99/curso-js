## 1. ¿Qué es una API?

Una API (Application Programming Interface) define reglas para que dos
sistemas intercambien información. Permite desacoplar cliente y servidor
mediante contratos claros de comunicación.

------------------------------------------------------------------------

## 2. ¿Qué es REST?

REST (Representational State Transfer) es un estilo arquitectónico para
diseñar servicios web sobre HTTP. Fue descrito por Roy Fielding y se
basa en restricciones que favorecen escalabilidad y simplicidad.

------------------------------------------------------------------------

## 3. Principios de REST

### Cliente-Servidor

Separación entre interfaz cliente y lógica del servidor.

### Stateless

Cada request contiene toda la información necesaria. El servidor no
guarda sesión entre peticiones.

### Cacheable

Las respuestas pueden almacenarse en caché mediante headers HTTP.

### Interfaz uniforme

-   Recursos identificados por URL
-   Uso semántico de HTTP
-   Representaciones como JSON

### Sistema en capas

Pueden existir proxies o gateways intermedios.

### Code on Demand (opcional)

El servidor puede enviar código ejecutable al cliente.

------------------------------------------------------------------------

## 4. Recursos y endpoints

Ejemplos:

/users\
/users/42\
/orders/15/items

Buenas prácticas: 
- Usar sustantivos 
- Evitar verbos en la URL 
- Mantener consistencia
-  Usar plural en recursos
-  Manejar errores de forma consistente
-  Implementar paginación
-  Documentar con OpenAPI

------------------------------------------------------------------------

## 5. Métodos HTTP

GET → obtener datos
POST → crear recursos
PUT → reemplazar recurso 
PATCH → modificación parcial
DELETE → eliminar recurso 

------------------------------------------------------------------------


## 7. Códigos de estado HTTP

2xx éxito: 
- 200 OK 
- 201 Created 
- 204 No Content

4xx error cliente: 
- 400 Bad Request 
- 401 Unauthorized 
- 404 Not Found

5xx error servidor: 
- 500 Internal Server Error 
- 503 ServiceUnavailable

------------------------------------------------------------------------

## 8. Formatos de datos

JSON es el formato más utilizado.

Ejemplo:

``` json
{
  "id": 1,
  "name": "Juan",
  "email": "juan@example.com"
}
```

Headers comunes:

Content-Type: application/json\
Accept: application/json

------------------------------------------------------------------------

## 9. Versionado de API

Opciones:

/api/v1/users\
/api/v2/users

O mediante headers personalizados.

------------------------------------------------------------------------

## 10. Seguridad

REST no define autenticación, pero se usan:

-   JWT
-   OAuth 2.0
-   API Keys
-   Basic Auth
